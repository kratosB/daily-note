# 线程池拒绝策略

## JDK内置4种线程池拒绝策略

1. CallerRunsPolicy（调用者运行策略）

   如果此时线程池没关，由提交的这个线程处理这个任务。
2. AbortPolicy（中止策略）

   直接抛出错误
3. DiscardPolicy（丢弃策略）

   把这个任务丢了（感觉不太好，只适用于某些场景，例如每隔五分钟统计，这次没统计，下次也会把这次的一并统计了）
4. DiscardOldestPolicy（弃老策略）

   跟上面那个一样，不过丢的是最老的一个。
